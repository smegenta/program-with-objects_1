Intro: Make your web page

The Basics of the Web:

Components of the web

The web is comprised of the following pieces:

  • Web Browser - A disply program on a computer to display content.
  • Servers - Computers that host files.
  • Http - Communication protocol.
  • Internet - A network of computers.
HTML

HTML is the heart of the web and is composed of the following parts:

  • Text - This is basically what gets displayed.
  • Markup - This controls the style of the text that is diaplyed.

An HTML may also contain references to other kinds of documents such as images and video. It may also contain links to other pages.

Markup

A Markup element is made of an opening tag, a content and closing tags. Opening tags may have values. Some markups do not have contants and are said to be void markups. Void tags do not need closing tag.

Some example of tags:

  • <a> - anchor for making links
  • <img> - for images and has src and alt attributes
  • <br> - page break
  • <p> - make paragraphs

The difference between the line break and paragraph tag is that the first one is an inline tag while the latter is a block tag.

HTML Documents

An HTML document has two parts: A head and body.

  • The Head contains meta data about the document including the title of the document.
  • The Body is basically where the content of the document is written.

Creating a structured Document with HTML

Conceptual description of a website

A website has a structure, a style and interactive components

  • The structure is done using HTML language
  • The style is done using CSS language
  • The interactive part is done using Java script.
  • Both HTML and CSS have their own Sytax and rules.
  • The tree structure of HTML follows the DOM.
Boxing design

All elements of a website are in boxes. The first thing you need to do when you start converting a graphic design into a webpage is to decide the boxes that will need to be created. Afterwards you populate the boxes with content.

Boxes are created with the div tag. To help add style to the boxes, include a class attribute to each box. The names should be indicative of the purpose of the box to avoid confusion lateron.

Adding CSS Style to HTML structure

What is CSS?

CSS stands for Cascading Style Sheets. It is used to control the style of the HTML elements using selectors which can either be defined inside the HTML or in a separate file.

The basic advantage of CSS is that it helps us to avoid repition, which helps to make the code good and concise.

Code-Refine-Test

Whenever you write code, first start with the big picture and drill down to the details. Always look for repeated styles to avaoid repition.

After witing the code, test in different browsers, resize browser to see the effect of on your coode and fix problems you see. Repeat the steps as necessary until all issues are fixed.

The If statement is used to make a decision based on comparison. The syntax of an If statement is

The IfTestExp part is evaluated first and if true block is executed.

OR operator

The OR operator is a logical construct which evalues to True if either the left ot right operand evalues to True.

The OR operator evalues only what it needs to evaluate, which means if the left operand evalues to True, it does not evaluate the right operand.

While Loop

Loop is a construct that does things over and over again based on a set criteria

The While loop has theollowing syntax:

Block

Block will be executed repeatedly as long as the WhileTestExp is true. Exceution will exit from the loop when the WhileTestExp evaluates to false.

Break Statements

A Break statement is used to break in the middle of a loop

Introduction

In this section of the course we learn about Functions, Classes and Object Oriented Programming. We will do so by building three projects: Break Scheduler, Profanity Editor and Movie Database. Object Oriented Programming is a programming paradigm used by most programmers

Procedural vs. Object Oriented Programming

Procedural Programming

Procedural programming is a paradigm where a program is written explictly as a sequence of steps to be followed to produce a result. To make programming easier, a prolem is broken down into smaller logical pieces and procedures/functions are written for each piece. The procedures/functions have no intrinsic relationship with the data they operate on. Data is separate from fuctions. Examples of procedural programming languages include C and Pascal.

Object oriented Programming

In object oriented programming, data and related functions are bundled together to form objects with the data in the objected accessed or manipulated by functions of the object. The objects are used to model real-world objects (such as a person). Objects can have relationship with other objects.(A parent object will have son objects and so on). programs are designed such that objects interact with one another. Examples of Object Oriented programming languages include Python, C++ and Java.

The first step in OOP is to identify all the objects a programmer wants to manipulate and how they relate to each other. Once an object has been identified, it is generalized as a class of objects that defines the kind of data it contains and functions(methods) that can manipulate it. Objects communicate with messages.

Using Functions

Abstractions

Python has an extensive list of built-in fuctions that we can use by simply calling them. Since these functions are already included in Python, we do not need to define them, nor are we concerned with the details of their implementation. To use them, we only need to know what they do, their inputs and their outputs. Examples of these type of functions include string functions such as len(), split() and so on.

The ability to use functions without bothering about the details of their implementation is called abstraction.

An example of an Abstraction is the use of CSS tags in HTML. Once a CSS tag is defined in the CSS file, we are not concerned about the definition when using it. We refernce it in the header of the HTML file and use the tags that are defined in CSS file in the body of the HTML file.

Advanced Topics

Instance vs. Class variables

Class variables are defined outside of the methods of the class and can be directly referenced from the class. Class variables can be useful for constants that will need to be used by all instances of the class, or that are meant to be accessed directly from the class. A good example is the movie ratings of the Movie class we saw during the vide lecture.

Instance variables are defined inside __init__ (), and they are local to an instance.

Pre-existing Class variables

Python comes with pre-existing variables for classes. Some of them are:

__doc__: When we define a class we can also write the documentation for them in the body of the class definition as a string surrounded by three double quotes.

__module__: This is the name of the module in which the class is defined.

__name__: Every Python module/program has a variable defined called __name__ which is set to '__main__' when the program is run by itself. If it is imported to another program, it is set to the name of the module

Inheritance

Inheritance is used to indicate that one class will get most or all of its features from another parent class. if you define a class as

class Child(Parent)

It means the class Child will inherit from class Parent

When the constructor for the child class is invoked, it in turn will invoke the constructor of the parent class. The members of the child class that are already defined in the parent class get initialized by the parent constructor. Then members of the class that are not initialized by the prent class constructor will be initialized by the child class constructor.

Method Overriding

In Inheritance, Child classes not only inherit the variables of the parent class but it also inherits the parent's methods. So even if a method is not explicitly defined in a child class, as long as it is defined in the parent class, it can be called from instances of the child class.

A method that is defined in a parent class can also be re-defined in the child class. This is usually done when the Child class implements the method in a different way. In this case when an instance of the parent class call the method, the version defined in the parent class will be executed, while if an instance of the child class calls the method, the version defined in the Child class will be called.

This property of redifing the method of a parent class in a child class us called method overriding.
